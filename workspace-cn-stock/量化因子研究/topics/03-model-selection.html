1. LightGBM（GBDT 家族）优势场景

在工程因子（Alpha158）上，树模型显著优于深度学习。LightGBM 的 IC 达到 0.0448，而 GRU 仅 0.0315。在 CSI500 上差距更大：LightGBM(0.0399) vs ALSTM(0.0229)。

  • 训练速度快：分钟级完成训练，DL 则需要小时级
  • 调参直观：核心参数（num_leaves, learning_rate, num_boost_round）含义清晰
  • 可解释性好：Feature importance 直接输出有效因子排序
  • 对缺失值天然鲁棒：无需预处理，树模型自带缺失值处理逻辑
  • 截面预测更擅长：在一个时间点对所有股票排序时，工程因子+LightGBM 是最优解
实战结论 如果你只用 Alpha158 类工程因子，LightGBM 几乎一定是最优选择。深度学习在此场景下的优势无法体现。

2. 深度学习优势场景

当输入从工程因子切换为原始序列数据（Alpha360：60天 x 6 OHLCV）时，深度学习反超：ALSTM IC 0.0497 vs LightGBM 0.0400。

  • 时序模式捕获：LSTM/GRU 能学习多日价量交互模式（如"缩量回踩后放量突破"）
  • 图神经网络（HIST）：通过行业/概念图共享信息，Rank IC 达 0.067
  • Transformer（MASTER）：市场引导注意力机制，组合年化+47%
  • 自动特征提取：从 raw data 中端到端学习，减少手工因子构造
关键洞察：特征决定模型 不是"LightGBM vs DL"的问题，而是"工程因子 vs 原始序列"的问题。Alpha158 上选树模型，Alpha360 上选 DL。两者做 ensemble 才是最优策略。

Q10: 怎么做滚动训练避免过拟合？

静态训练（train once, predict forever）是量化新手最常见的错误。金融数据非平稳（non-stationary），模型的预测能力会随时间衰减。滚动训练是应对这一问题的核心手段。

1. 为什么需要滚动训练

  • 数据非平稳：2017 年有效的模式，到 2024 年可能已完全失效
  • 概念漂移（Concept Drift）：市场结构、参与者构成、监管政策持续变化
  • IC 时间衰减：静态模型的样本外 IC 通常每年衰减 10-20%
  • 前沿研究：DoubleAdapt (KDD 2023) 和 AdaRNN 专门解决分布漂移问题
常见误区 "我用了 2010-2023 全部数据训练，数据量够大，应该够泛化了。" ——错。数据量大但分布不稳定时，模型会学到过时的信号。用全量数据训练的 LightGBM，在 2024 年的 IC 可能仅为训练期的 60%。

2. 三种训练窗口策略

a) 固定窗口滚动（Rolling Window）

训练窗口固定为 N 年，每到重训节点就向前滚动。旧数据被"遗忘"，始终只用最近 N 年。

Train: [t-N, t-gap] → Valid: [t-gap, t] → Predict: [t, t+step]
下一轮: Train: [t-N+step, t+step-gap] → Valid: [t+step-gap, t+step] → Predict: [t+step, t+2*step]
  • 优点：自动遗忘过时数据，对概念漂移适应性强
  • 缺点：丢弃可能仍有用的早期数据
  • 适合：短期因子（反转、短期动量、量价因子）
  • 推荐窗口：3-5 年训练期

b) 扩展窗口（Expanding Window）

训练窗口起点固定，终点不断延伸。随着时间推移，训练数据量持续增长。

Round 1: Train [2008, 2020] → Valid [2020, 2021] → Test [2021]
Round 2: Train [2008, 2021] → Valid [2021, 2022] → Test [2022]
Round 3: Train [2008, 2022] → Valid [2022, 2023] → Test [2023]
  • 优点：数据量大，信号更稳定，不浪费历史数据
  • 缺点：近期数据占比越来越小，早期噪声累积
  • 适合：价值、质量等长期稳定因子

c) 混合策略（Hybrid / Sample Weighting）

保留全部历史数据，但对近期数据赋予更高权重。兼顾数据量和时效性。

数据时段 距今时间 样本权重 说明
近 1 年 0-12 月 1.0 最高权重，反映当前市场
1-3 年 12-36 月 0.7 中等权重
3-5 年 36-60 月 0.4 较低权重
5 年以上 60+ 月 0.2 仅保留长期规律
DoubleEnsemble 的内置机制 DoubleEnsemble 使用类似的学习轨迹重加权（sample reweighting by learning trajectory），自动为"难学"样本分配更高权重，类似 AdaBoost 思想。无需手动设定权重衰减。

3. 重训频率选择

频率 计算成本 适应性 推荐场景 备注
每日 极高 最强 高频策略 / 生产环境 需自动化 pipeline
每周 高 强 短期因子 周末跑训练
每月 中等 中 推荐起步方案 月初重训，当月预测
每季度 低 较弱 稳定因子 / 低频策略 配合财报季
每半年 最低 弱 仅基本面因子 最低限度更新

以下是 Python 代码中使用滚动训练的完整流程：

import qlib
from qlib.workflow import R
from qlib.model.trainer import TrainerR
from qlib.workflow.task.gen import RollingGen
from qlib.workflow.task.collect import RecorderCollector
from qlib.workflow.record_temp import SignalRecord, SigAnaRecord

# Step 1: 初始化 Qlib
qlib.init(provider_uri="~/.qlib/qlib_data/cn_data", region="cn")

# Step 2: 定义基础任务（单个时间窗口）
task = {
    "model": {
        "class": "LGBModel",
        "module_path": "qlib.contrib.model.gbdt",
        "kwargs": {"loss": "mse", "num_leaves": 210},
    },
    "dataset": {
        "class": "DatasetH",
        "module_path": "qlib.data.dataset",
        "kwargs": {
            "handler": {
                "class": "Alpha158",
                "module_path": "qlib.contrib.data.handler",
            },
            "segments": {
                "train": ("2008-01-01", "2017-12-31"),
                "valid": ("2018-01-01", "2019-12-31"),
                "test":  ("2020-01-01", "2020-12-31"),
            },
        },
    },
}

# Step 3: 用 RollingGen 生成多个滚动任务
rg = RollingGen(
    step=20,              # 每20个交易日滚动一步
    rtype="rolling",     # "rolling" or "expanding"
)
tasks = rg(task)        # 返回 list of tasks，每个对应一个时间窗口

# Step 4: 逐个训练并记录预测结果
trainer = TrainerR(experiment_name="rolling_lgbm")
for t in tasks:
    with R.start(experiment_name="rolling_lgbm"):
        model = trainer.train(t)
        recorder = R.get_recorder()
        # 记录预测信号
        sr = SignalRecord(model, t["dataset"], recorder)
        sr.generate()
        # 记录信号分析（IC等指标）
        sar = SigAnaRecord(recorder)
        sar.generate()

# Step 5: 收集所有滚动窗口的预测，拼接为完整时间序列
collector = RecorderCollector(experiment_name="rolling_lgbm")
predictions = collector.collect()
print(f"Total predictions shape: {predictions.shape}")

5. 验证集设计要点

验证集的设计对防止过拟合至关重要。在时序数据中，验证集必须遵守严格的时间顺序。

设计要点 推荐做法 常见错误
时间顺序 验证集严格在训练集之后 随机 shuffle 打乱时序
Gap 设置 训练集与验证集之间留 5 天 gap 训练集和验证集紧邻
验证集用途 仅用于 early stopping + 超参选择 用验证集 IC 选特征
测试集隔离 测试集完全不参与任何决策 根据测试集结果调参
验证集比例 训练:验证 = 4:1 或 5:1 验证集过大浪费数据
数据泄露的隐蔽来源 即使做了时间顺序分割，以下操作仍可能导致数据泄露：(1) 用全量数据做标准化（应只用训练集计算均值/标准差）；(2) 因子构造用了未来函数（如收盘价对齐到当日而非次日）；(3) 行业分类用了最新版而非历史版。

6. 过拟合信号检测

如何判断模型是否过拟合？以下信号是明确的警告：

过拟合信号 具体表现 诊断阈值 应对措施
IC 逐级衰减 训练IC >> 验证IC >> 测试IC 衰减超过 30% 增加正则化，减少模型复杂度
换手率异常高 月度换手率 > 200% 对比基准策略换手率 添加换手率惩罚，降低预测灵敏度
时间不稳定 近 1-2 年 IC 显著低于早期 近期IC < 历史IC的 50% 缩短训练窗口，增加重训频率
因子集中度 Feature importance 高度集中在少数因子 Top3 因子占比 > 60% 特征子采样 (colsample_bytree)
过拟合验证集 验证集IC高但测试集IC低 验证IC/测试IC > 1.5 更换验证集时间段，交叉验证 
# 过拟合诊断代码片段
import pandas as pd
import numpy as np

def diagnose_overfitting(predictions, labels, dates):
    """
    对滚动预测结果进行过拟合诊断
    predictions: pd.Series, 模型预测值
    labels: pd.Series, 实际收益率
    dates: pd.Series, 日期索引
    """
    df = pd.DataFrame({
        "pred": predictions,
        "label": labels,
        "date": dates,
    })

    # 1. 分时段计算IC
    monthly_ic = df.groupby(df["date"].dt.to_period("M")).apply(
        lambda x: x["pred"].corr(x["label"])
    )

    # 2. 检测近期IC是否衰减
    recent_ic = monthly_ic[-12:].mean()
    historical_ic = monthly_ic[:-12].mean()
    decay_ratio = recent_ic / historical_ic

    print(f"历史IC均值: {historical_ic:.4f}")
    print(f"近12月IC均值: {recent_ic:.4f}")
    print(f"衰减比例: {decay_ratio:.2f}")

    if decay_ratio < 0.5:
        print("WARNING: 严重过拟合信号！近期IC不足历史的50%")
    elif decay_ratio < 0.7:
        print("CAUTION: 中度衰减，建议缩短训练窗口")
    else:
        print("OK: IC衰减在可接受范围内")

    return monthly_ic, decay_ratio
最佳实践总结 推荐起步方案：每月滚动重训 + 3 年固定窗口 + 5 天 gap + LightGBM。这个配置在计算成本和适应性之间取得了最佳平衡，个人 PC 即可运行。

Q11: Ensemble 怎么做效果最好？

单一模型的预测能力终究有限。Ensemble（集成学习）通过组合多个模型的预测，能稳定提升 IC 和降低回撤。关键在于选择多样性高的 base model 组合。

1. 简单加权平均（Blending）

最简单也最鲁棒的 ensemble 方式。几乎没有过拟合风险，推荐所有人先尝试。

pred_final = w1 × pred_lgbm + w2 × pred_alstm
权重策略 公式 优点 缺点
等权 w = 0.5 / 0.5 最简单，零过拟合风险 忽略模型质量差异
验证集 IC 加权 wi = ICi / ΣIC 自动偏向更好的模型 轻微过拟合验证集
验证集 Sharpe 加权 wi = SRi / ΣSR 同时考虑收益和稳定性 需要回测，计算量大
滚动 IC 加权 wi,t = ICi,t-k:t / ΣIC 权重动态适应 参数选择（回看窗口 k） 
import pandas as pd
import numpy as np

def simple_blend(pred_lgbm, pred_alstm, method="equal", valid_ic=None):
    """
    简单加权 Ensemble
    pred_lgbm, pred_alstm: pd.Series，同index
    method: "equal" | "ic_weighted"
    valid_ic: dict, e.g. {"lgbm": 0.045, "alstm": 0.050}
    """
    if method == "equal":
        w1, w2 = 0.5, 0.5
    elif method == "ic_weighted":
        total = valid_ic["lgbm"] + valid_ic["alstm"]
        w1 = valid_ic["lgbm"] / total
        w2 = valid_ic["alstm"] / total

    pred_final = w1 * pred_lgbm + w2 * pred_alstm
    print(f"Blend weights: LGBM={w1:.3f}, ALSTM={w2:.3f}")
    return pred_final

# 使用示例
pred_ensemble = simple_blend(
    pred_lgbm, pred_alstm,
    method="ic_weighted",
    valid_ic={"lgbm": 0.045, "alstm": 0.050}
)
# 预期效果：Rank IC 提升约 5-10%
Blending 够用吗？ 对于个人量化来说，LGBM + ALSTM 等权混合已经能获得很好的效果。Rank IC 提升 5-10% 且完全不增加过拟合风险。如果时间有限，用 Blending 即可。

  • 样本重加权：给"难学"的样本更高权重（类似 AdaBoost 思想），聚焦难预测的时间段

Stacking 是更高级的 ensemble 方法：Level 0 的多个 base model 各自预测，然后将这些预测作为特征，训练 Level 1 的 meta model。

import numpy as np
import pandas as pd
from sklearn.linear_model import Ridge

def time_series_stacking(base_preds, labels, dates, n_splits=5):
    """
    时序 Stacking 框架
    base_preds: dict of pd.Series, {"lgbm": pred1, "alstm": pred2, "hist": pred3}
    labels: pd.Series, 实际收益率
    dates: pd.Series, 日期
    n_splits: 时序分割数
    """
    # 构建 meta features DataFrame
    meta_features = pd.DataFrame(base_preds)
    unique_dates = sorted(dates.unique())
    split_size = len(unique_dates) // (n_splits + 1)

    oof_predictions = pd.Series(index=labels.index, dtype="float64")

    # 时序交叉验证（不打乱时间顺序！）
    for i in range(n_splits):
        train_end = unique_dates[(i + 1) * split_size]
        valid_start = unique_dates[(i + 1) * split_size + 5]  # 5天gap
        valid_end = unique_dates[(i + 2) * split_size]

        train_mask = dates <= train_end
        valid_mask = (dates >= valid_start) & (dates <= valid_end)

        # 训练 meta model (Ridge回归，防过拟合)
        meta_model = Ridge(alpha=1.0)
        meta_model.fit(
            meta_features[train_mask],
            labels[train_mask]
        )

        # out-of-fold 预测
        oof_predictions[valid_mask] = meta_model.predict(
            meta_features[valid_mask]
        )

    return oof_predictions

# 使用示例
stacked_pred = time_series_stacking(
    base_preds={
        "lgbm": pred_lgbm,
        "alstm": pred_alstm,
        "hist": pred_hist,
    },
    labels=actual_returns,
    dates=trade_dates,
)
# 预期效果：Rank IC 提升约 10-15%
Stacking 的数据泄露陷阱 必须使用 out-of-fold 预测作为 meta features。如果直接用 base model 在训练集上的预测来训练 meta model，会导致严重过拟合。在时序场景下，用时序 split 替代 K-fold。

4. 模型多样性原则

Ensemble 的关键不是模型越多越好，而是 base model 之间的预测相关性要低。最佳组合是让每个模型捕获不同类型的信号。

预测相关性矩阵示例

LightGBM (A158) ALSTM (A360) HIST (A360)
LightGBM (A158) 1.00 0.45 0.38
ALSTM (A360) 0.45 1.00 0.72
HIST (A360) 0.38 0.72 1.00
  • LightGBM vs ALSTM 相关性 0.45：很低，ensemble 效果好（不同特征集 + 不同模型结构）
  • ALSTM vs HIST 相关性 0.72：较高，因为共用 Alpha360 特征集
  • 最佳组合：Tree(Alpha158) + Sequential(Alpha360) + Graph(HIST)，三者分别捕获截面排序 + 时序模式 + 股票间关系
多样性 > 单体性能 两个 IC 为 0.04 但相关性 0.3 的模型做 ensemble，效果好于两个 IC 为 0.05 但相关性 0.8 的模型。选择 base model 时，优先保证多样性。

6. 时序 Ensemble 特殊考虑

因子和模型的效果会随时间变化，固定权重的 ensemble 无法适应这种变化。动态权重调整是进阶方案。

动态权重方案

  • 滚动 IC 加权：用最近 3-6 个月的滚动 IC 作为权重，自动偏向近期表现好的模型
  • AlphaForge 方法：每个时间步基于 meta-learning 动态重加权
  • 简单 Regime 切换：牛市偏重动量模型，震荡市偏重反转模型
def dynamic_ensemble(preds_dict, labels, dates, lookback=60):
    """
    动态 IC 加权 Ensemble
    preds_dict: {"model_name": pd.Series}
    labels: pd.Series
    dates: pd.DatetimeIndex
    lookback: 回看天数（计算滚动IC用）
    """
    import pandas as pd
    import numpy as np

    all_dates = sorted(dates.unique())
    result = pd.Series(index=labels.index, dtype="float64")

    for i, date in enumerate(all_dates):
        if i < lookback:
            # 回看期不够，用等权
            weights = {k: 1.0 / len(preds_dict) for k in preds_dict}
        else:
            # 计算每个模型近 lookback 天的 IC
            recent_dates = all_dates[i - lookback:i]
            mask = dates.isin(recent_dates)
            rolling_ics = {}
            for name, pred in preds_dict.items():
                ic = pred[mask].corr(labels[mask])
                rolling_ics[name] = max(ic, 0.001)  # 下限防止负权

            total = sum(rolling_ics.values())
            weights = {k: v / total for k, v in rolling_ics.items()}

        # 加权融合当天预测
        day_mask = dates == date
        for name, pred in preds_dict.items():
            result[day_mask] += weights[name] * pred[day_mask]

    return result
动态权重的陷阱 回看窗口太短（如 20 天）会导致权重剧烈波动，实际效果反而不如等权。建议回看窗口至少 60 个交易日（约 3 个月），并设置权重变化幅度上限。
