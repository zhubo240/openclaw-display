Q16: 回测到实盘的衰减有多大？

回测表现永远比实盘好。理解衰减的来源和幅度，是设定合理盈利预期的前提。如果回测就勉强跑赢基准，那实盘大概率亏钱。

1. 衰减幅度的行业经验

实盘表现 ≈ 回测表现 × 0.4 ~ 0.7

这个衰减系数因团队基础设施水平而异：

0.6-0.7
百亿私募衰减系数
基础设施好、执行成本低
0.4-0.6
个人量化衰减系数
执行差、成本高、数据质量低
6-10%
回测15% → 实盘预期
个人量化预期范围
2-5%
回测8% → 实盘预期
可能不如买指数基金
核心判断标准 如果回测年化低于 15%，个人量化实盘后很可能跑不赢沪深300。回测年化 8% 的策略，扣除衰减后实盘只剩 2-5%，还不如买余额宝。

2. 衰减来源分解

衰减不是一个黑箱，可以拆解为几个可量化的来源：

来源占衰减比例说明
交易成本低估30-40%冲击成本、滑点在回测中难以精确模拟
信号延迟15-20%回测假设收盘价下单，实盘有执行延迟
过拟合20-30%模型学到了历史噪声，未来无法重现
数据质量差异10-15%回测用清洗后数据，实盘数据有毛刺
市场冲击5-10%自身交易影响价格（百万级可忽略）

3. 如何在回测中减少衰减

在回测阶段就做好保守假设，可以大幅缩小回测与实盘的差距：

3.1 交易成本设置

# 保守的交易成本模型
cost_config = {
    "commission": 0.0003,     # 佣金 0.03%（双边）
    "stamp_tax": 0.0005,      # 印花税 0.05%（卖出单边）
    "slippage": 0.001,        # 滑点 0.1%
    "impact": 0.001,          # 冲击成本 0.1%（百万级可低些）
}
# 单次交易总成本 ≈ 0.25-0.35%
# 月换手 30% → 月交易成本 ≈ 0.15-0.21%

3.2 下单假设

  • 不要用当日收盘价下单 — 实盘中收盘价出来时已经收盘了
  • 推荐：用次日开盘价或次日 VWAP 作为成交价假设
  • Qlib 中设置 trade_start_time="09:30"，trade_end_time="15:00"

3.3 可交易性过滤

def filter_tradable(df):
    """剔除不可交易的股票"""
    mask = (
        (df["$close"] != df["$limit_up"]) &   # 非涨停
        (df["$close"] != df["$limit_down"]) & # 非跌停
        (~df["$suspended"]) &                  # 非停牌
        (df["$turnover"] > 5_000_000)           # 日均成交额>500万
    )
    return df[mask]

3.4 多段验证与样本外测试

  • 多段验证：将历史数据分成 3-5 段，每段都要正收益才算通过
  • 样本外保留：最近 6-12 个月数据完全不用于训练和调参
  • 跨市场验证：如果在沪深300上有效，在中证500上也应该有效（方向一致）

Q17: 百万级资金的容量约束在哪里？

策略容量是指在不显著影响市场价格的前提下，能管理的最大资金量。好消息是：对百万级个人资金来说，几乎不存在容量问题。

1. 容量的定义

策略容量 = 持仓只数 × 单只股票可交易额 × (1 / 换手频率)

关键规则：单只股票的每日交易量不超过其日均成交额的 1%（1% ADV rule），以避免对价格造成显著冲击。

2. A股各板块流动性

股票池日均成交额（中位数）1% ADV 上限10只组合月容量
沪深300~15亿~1500万~3亿
中证500~5亿~500万~1亿
中证1000~2亿~200万~4000万
全A小盘~5000万~50万~1000万

3. 百万级资金的分析

以 100 万资金、持仓 15 只股票为例：

每只股票持仓
100万 / 15 只
0.003%

6.7万 / 2亿

百万级资金结论 即使在全A最小盘的股票中，百万级资金的单笔交易只占日均成交额的 0.01% 量级，完全可以忽略市场冲击。百万级资金在中证500、中证1000甚至小盘股中都没有容量约束。真正的约束在千万级以上才开始出现。

5. 容量约束的实际影响模型

def estimate_impact_cost(trade_amount, adv, sigma_daily=0.02):
    """
    估算市场冲击成本（Square-Root Model）
    trade_amount: 交易金额
    adv: 日均成交额
    sigma_daily: 日波动率
    """
    participation_rate = trade_amount / adv
    impact = sigma_daily * (participation_rate ** 0.5)
    return impact

# 百万级: impact = 0.02 * (67000/200000000)^0.5 = 0.0004 (0.04%) 可忽略
# 千万级: impact = 0.02 * (670000/200000000)^0.5 = 0.0012 (0.12%) 需关注
# 亿级:   impact = 0.02 * (6700000/200000000)^0.5 = 0.0037 (0.37%) 显著
个人量化的真正优势 百万级资金反而是优势 — 你可以在小盘股中挖掘 alpha，这些股票大资金根本进不去。很多 alpha 信号在小盘股中更强，因为机构覆盖不足、定价效率低。不要急于做大规模，先跑通策略逻辑再说。

4. 预测 + 组合指令生成

# daily_predict.py -- 加载模型 → 预测 → TopkDropout → CSV
import pandas as pd, joblib, qlib
from qlib.data import D

qlib.init(provider_uri="~/.qlib/qlib_data/cn_data")
model = joblib.load("models/lgbm_ensemble_latest.pkl")
today = pd.Timestamp.now().strftime("%Y-%m-%d")

# Step 1-2: 取特征 + 预测
features = D.features(D.instruments("csi500"),
    fields=["$close/$close[5]-1", "$volume/$volume[20]", ...],
    start_time=today, end_time=today)
scores = pd.Series(model.predict(features), index=features.index).sort_values(ascending=False)

# Step 3: TopkDropout -- 剔除持仓中得分最差的n_drop只，补入新的top
TOPK, N_DROP = 15, 3
current = set(pd.read_csv("state/holdings.csv")["code"])
drop = scores[scores.index.isin(current)].tail(N_DROP).index.tolist()
keep = current - set(drop)
new_buys = scores[~scores.index.isin(keep)].head(TOPK - len(keep)).index.tolist()

# Step 4: 输出交易指令CSV
orders = ([{"code": c, "action": "SELL"} for c in drop]
        + [{"code": c, "action": "BUY"}  for c in new_buys])
pd.DataFrame(orders).to_csv(f"orders/orders_{today}.csv", index=False)

Q19: 怎么监控策略是否失效？

策略上线不是终点，而是另一个起点。市场在变化，因子在衰减，任何策略都有生命周期。建立有效的监控体系，才能在策略失效前及时发现问题。

1. IC 滚动跟踪

IC（Information Coefficient）是策略预测能力的核心度量。实盘中需要持续跟踪滚动IC的变化趋势：

监控窗口用途预警阈值
20日滚动IC短期信号质量< 0.01 → 橙色预警
60日滚动IC中期趋势判断< 0 → 红色预警，考虑暂停
120日滚动IC长期基准对比< 历史均值 - 2σ → 策略可能失效
 
def rolling_ic(pred, ret, windows=[20, 60, 120]):
    """计算滚动 Rank IC 和 ICIR"""
    daily_ic = pred.groupby(level=0).apply(
        lambda x: x.corr(ret.loc[x.index], method="spearman")
    )
    result = {}
    for w in windows:
        result[f"IC_{w}d"] = daily_ic.rolling(w).mean()
        result[f"ICIR_{w}d"] = daily_ic.rolling(w).mean() / daily_ic.rolling(w).std()
    return pd.DataFrame(result)

def check_ic_alerts(ic_df, hist_mean, hist_std):
    """IC 预警: 橙=短期弱, 红=中期归零/长期偏离"""
    alerts, L = [], ic_df.iloc[-1]
    if L["IC_20d"] < 0.01:  alerts.append(("ORANGE", "20日IC<0.01"))
    if L["IC_60d"] < 0:     alerts.append(("RED", "60日IC归零!"))
    if L["IC_120d"] < hist_mean - 2*hist_std:
        alerts.append(("RED", "120日IC低于历史-2σ!"))
    return alerts

2. 因子拥挤度监控

当太多资金追逐同一个因子时，因子就会变得拥挤，alpha 被压缩甚至反转。

2.1 拥挤度信号

  • 因子估值价差：当前 top-bottom 组的估值价差 vs 历史分位数。如果价差处于历史极端（如 >95% 分位），说明因子已经非常拥挤
  • 因子 ETF 资金流入：同类因子 ETF（如红利 ETF、价值 ETF）AUM 快速增长 → 拥挤信号
  • 策略回撤相关性：多个独立策略同时出现回撤 → 系统性拥挤

2.2 2024 量化踩踏的教训

实战案例：2024年初量化踩踏 大量量化私募DMA策略集中在小盘风格，流动性枯竭时形成踩踏。早期信号：小盘因子价差达历史极端、多产品净值高相关回撤、监管关注DMA杠杆。个人量化规模小不会被踩踏，但持仓与机构高度重叠仍会受连带影响。 
def monitor_crowding(spread_hist, current):
    """因子拥挤度: 当前价差在历史中的分位数"""
    pctl = (spread_hist < current).mean()
    if pctl > 0.95: return "DANGER", f"价差 {pctl:.0%} 分位, 极度拥挤!"
    if pctl > 0.80: return "WARN", f"价差 {pctl:.0%} 分位, 需关注"
    return "OK", f"价差 {pctl:.0%} 分位, 正常"

def strategy_dashboard(stats):
    """策略健康度仪表盘 -- 返回红黄绿灯状态"""
    rules = {
        "ic_20d":     ("higher", 0.03, 0.01),  # (方向, 绿线, 黄线)
        "sharpe_60d": ("higher", 1.0,  0.3),
        "max_dd":     ("lower",  0.05, 0.10),
        "turnover_m": ("lower",  0.40, 0.60),
        "max_weight": ("lower",  0.15, 0.25),
        "sector_dev": ("lower",  0.10, 0.20),
    }
    return {k: classify(stats[k], *v) for k, v in rules.items()}

4. 什么时候该暂停策略

以下任一条件触发时，应认真考虑暂停策略并进行诊断：

触发1
连续3个月跑输基准
触发2
60日IC归零或转负
触发3
异常行为偏离回测
触发4
外部环境剧变
  • 连续3个月跑输基准：短期跑输正常，但3个月是一个有意义的窗口
  • 60日IC归零或转负：说明模型的预测能力已经消失
  • 异常行为偏离回测：如换手率突然飙升、持仓集中度异常等
  • 外部环境剧变：如 2024 年 DMA 限制、T+0 政策变化等监管层面的变化

5. 策略恢复决策流程

暂停策略后，需要系统化诊断和修复，不能简单等待恢复：

策略暂停 ── 诊断原因 │ ├── 因子失效？ ── 更新因子集，剔除无效因子，重新训练 ├── 模型过拟合？ ── 增加正则化，缩短训练窗口，减少特征 └── 市场regime变化？── 分析风格切换方向，调整参数适配 │ ├── 修复后: 重新跑样本外回测 + 1-3月模拟盘 └── 验证通过 ── 恢复实盘（从小资金开始）
def diagnose_failure(factor_ics, market_regime):
    """策略失效诊断: 逐因子检查IC衰减 + 市场环境判断"""
    issues = []
    for f, ic in factor_ics.items():
        if ic.tail(60).mean() < ic.mean() * 0.3:
            issues.append(f"因子[{f}]近60日IC严重衰减")
    if market_regime["vol_pctl"] > 0.9:
        issues.append("高波环境，策略可能不适配")
    if market_regime.get("style_shift"):
        issues.append(f"风格切换: {market_regime['style_shift']}")
    return issues or ["未发现明确原因，建议继续观察"]
策略监控的核心心态 不要因为一两周的回撤就惊慌失措，也不要因为"还没亏很多"就忽视预警信号。建立规则，按规则行事： (1) 黄灯 → 加强关注，检查原因
(2) 红灯 → 减仓或暂停，启动诊断
(3) 恢复 → 必须重新经过模拟盘验证
量化的优势就在于纪律性 — 不要让情绪覆盖规则。
实盘落地总结 (1) 回测到实盘的衰减通常在 30-60%，回测年化 < 15% 的策略实盘风险很大。
(2) 百万级资金几乎没有容量约束，反而是优势 — 可以在小盘股中挖掘 alpha。
(3) 自动化流程从"每日CSV + 手动下单"起步，验证稳定后再升级工程。
(4) 建立 IC 滚动跟踪 + 健康度仪表盘，用规则而非情绪来管理策略生命周期。
(5) 策略失效是常态，重要的是有系统化的诊断和恢复流程。
