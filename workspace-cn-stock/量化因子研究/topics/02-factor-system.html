Q4：A股哪些因子有持续超额收益？

A股市场的因子结构与美股有本质差异。散户交易占比约 60%、T+1 制度、涨跌停限制等微观结构特征，使得部分在美股失效的因子在 A 股持续有效，而美股经典的动量因子在 A 股反而失效。以下逐一分析 A 股最具 alpha 的因子。

为什么 A 股反转而非动量？ 美股 2-12 月动量因子（Jegadeesh & Titman 1993）是最经典的异象之一，但 A 股恰好相反：短期过去收益率与未来收益率负相关。根本原因是散户占比高导致过度反应，随后均值回归。

  • 短期反转（5日/10日/20日）：过去 N 日收益率越低，未来收益率越高。月度多空价差 1.0-1.5%
  • 与美股对比：美股 2-12 月动量显著（月度溢价约 1%），A 股恰好相反，动量因子在 A 股几乎无效甚至为负
  • 相关论文：Dania & Spillan 2019, Liu-Zhou-Zhu CH-4 模型中的 trend factor
风险警示：2024年量化踩踏中反转因子短期失效 2024年初微盘股踩踏期间，反转因子经历了罕见的失效。当市场出现系统性流动性危机时，均值回归逻辑不再成立，反而出现价格持续下跌的动量效应。这提醒我们即使最稳定的因子也需要风控机制。

高特质波动率的股票未来收益显著更低。这与 CAPM 的预测相矛盾（高风险应获得高回报），被称为"低波异象"（low volatility anomaly）。

  • 特质波动率通过对 Fama-French 三因子模型回归取残差的标准差计算
  • 可分解为"好"IVol（由正面消息驱动的波动）和"坏"IVol（由负面消息驱动的波动），坏 IVol 驱动负收益
  • 与低波因子协同但捕获不同维度：低波因子关注总波动率，IVol 关注剔除系统性风险后的残余波动
  • IC 为负值表示做空高 IVol + 做多低 IVol 获利，实际使用时需取负号

3. 换手率因子 (Turnover) — A股独有的稳定因子

低换手率股票持续跑赢高换手率股票。在 A 股市场中，高换手率往往反映投机行为而非信息驱动的交易。

  • 换手率 = 成交量 / 流通股本。A 股流动性充裕但分布极不均匀
  • 换手率波动率（turnover volatility）也有独立预测效果：换手率不稳定的股票未来表现更差
  • 高换手率往往伴随散户资金的大量进出，是投机情绪的直接代理变量
  • 数据需求：需要流通股本（float_share）数据做归一化，Alpha158 只有 volume 无 turnover

4. 价值因子 (Value) — 长期有效但周期性强

A 股价值因子有几个独特特征：

  • EP 优于 BP：CH-3 模型（Liu-Stambaugh-Yuan 2019）使用 EP（earnings-price ratio）作为 value proxy，效果优于传统的 BP（book-to-price）
  • 年化溢价约 4-6%，但周期性极强：2017-2020"核心资产"期间严重失效（大盘成长风格碾压价值）
  • 2021 年后强势回归：红利/价值风格重新受到市场追捧，叠加利率下行环境
  • 需要行业中性化处理：银行 PE 约 5，科技 PE 约 50，不同行业估值中枢差异巨大
价值因子的周期性 价值因子的周期性意味着它不适合作为单一策略使用。最佳实践是与质量因子组合：价值负责进攻（在价值回归期获取超额），质量负责防守（在价值失效期控制回撤）。

5. 质量因子 (Quality) — 最稳定的防御性因子

  • 核心指标：ROE、毛利率、盈利稳定性（earnings volatility 的倒数）
  • 稳定性最好：各种市场环境下都不差，但爆发力不强
  • 月均溢价 0.4-0.5%，年化约 5-6%
  • 与价值因子互补：质量因子在成长股行情中也能提供正超额（好公司总是被追捧），而价值因子在此环境下会失效

SMB（Small Minus Big）是 A 股最显著的因子之一。小市值股票长期跑赢大市值股票，溢价来源包括壳价值、信息不对称、散户偏好等。

微盘股风险：2024 年踩踏教训 2024 年初微盘股指数一周暴跌超过 20%。微盘股（市值 < 30 亿）流动性极差，一旦出现集中卖出，价格可以连续跌停。建议保留小盘暴露但严格剔除微盘股。注册制 + 退市常态化正在削弱传统的壳价值溢价。

7. 涨跌停因子 (Price Limit) — A股特有

  • 频繁触涨停的股票未来收益显著更低（月度约 -0.76%），体现投资者过度反应
  • 涨停板制度（±10%/20%）创造了强制隔夜持有，放大了投资者的过度反应
  • 可以作为反转因子的增强维度：近期涨停次数越多，反转信号越强
  • 这是纯粹由 A 股制度造成的因子，在没有涨跌停限制的市场中不存在

Q5：Alpha158 具体有哪些弱点，如何改造？

Alpha158 是 Qlib 内置的因子集，包含 158 个基于 OHLCV（开高低收量）数据的技术因子。它是一个通用起点，但针对 A 股市场存在明显的结构性缺陷。

1. Alpha158 的 158 个因子结构

类别因子数量说明A股有效性
KBAR 9 开高低收关系：(close-open)/open, (high-low)/open 等 中等
PRICE 5 归一化价格，5/10/20/30/60日 lookback 中等
VOLUME 5 归一化成交量，5/10/20/30/60日 lookback 中等
ROLLING ~139 27+ 基础因子 × 5 个滚动窗口（5/10/20/30/60日） 参差不齐
核心问题：全部基于 OHLCV，无基本面数据 Alpha158 的 158 个因子完全基于价格和成交量数据，没有任何基本面因子（ROE、PE、营收增速等），也没有 A 股特有的市场微观结构因子（换手率、涨跌停、北向资金等）。

2. 逐类别诊断

KBAR 因子（9个）

捕获日内价格形态（如上影线、下影线、实体大小）。在 A 股中效果中等，因为日内形态受涨跌停限制影响较大。部分因子（如 KMID、KLEN）有微弱的预测力，但 IC 通常在 0.02-0.03 之间。

PRICE 因子（5个）

本质上是对不同窗口价格动量/反转的间接捕获。窗口固定（5/10/20/30/60日）是其局限，缺少超短期（1-3日）和中期（90/120日）窗口。在 A 股中，5日和10日窗口的反转效果最强。

VOLUME 因子（5个）

归一化成交量有一定效果，但关键缺陷是：有 volume 但无 turnover。成交量受股本大小影响，不同股票之间不可比。需要用流通股本做归一化才能得到真正的换手率因子。

ROLLING 因子（~139个）

覆盖面广（std、mean、max、min、correlation、regression 等），但很多因子的 IC 接近 0。这些噪声因子会干扰模型学习，尤其对树模型的特征选择产生误导。

Q6：怎么构造 A 股特有因子？

以下为每个 A 股特有因子提供具体计算公式、Qlib 表达式、Python 实现代码和预期 IC 范围。

1. 短期反转因子

REV_N = -1 × (Pt / Pt-N - 1) = Ref($close, N) / $close - 1
因子Qlib 表达式预期 RankIC说明
REV_5 Ref($close, 5) / $close - 1 5-8% 5日反转，最经典的 A 股 alpha
REV_10 Ref($close, 10) / $close - 1 4-7% 10日反转
REV_20 Ref($close, 20) / $close - 1 3-5% 20日反转（月度）

注意：公式中 Ref($close, N) 取的是 N 日前的收盘价。当过去收益为负时，因子值为正，表示预期反转上涨。无需额外乘以 -1，因为公式本身已包含反转逻辑。

2. 换手率因子

TURN = Volume / Float_Share      TURN_N = Mean(TURN, N) 
# 换手率因子 - Python 实现
import pandas as pd
import numpy as np

# 数据源: Tushare daily_basic 字段 float_share (万股)
def calc_turnover_factors(df):
    """
    df 需要包含: volume (手), float_share (万股)
    """
    # 日换手率 = 成交量(手) / 流通股本(手)
    df['turnover'] = df['volume'] / (df['float_share'] * 10000)

    # 20日平均换手率
    df['TURN_20'] = df.groupby('stock_id')['turnover'].transform(
        lambda x: x.rolling(20).mean()
    )

    # 换手率波动率 (独立效果)
    df['TURN_VOL'] = df.groupby('stock_id')['turnover'].transform(
        lambda x: x.rolling(20).std()
    )

    # 异常换手率 = 短期 / 长期 - 1
    turn_5 = df.groupby('stock_id')['turnover'].transform(
        lambda x: x.rolling(5).mean()
    )
    turn_60 = df.groupby('stock_id')['turnover'].transform(
        lambda x: x.rolling(60).mean()
    )
    df['TURN_ABNORMAL'] = turn_5 / turn_60 - 1

    return df

3. 涨跌停因子

LIMIT_HIT_UP_N = Σi=1..N I(closei ≥ high_limiti)      DIST_TO_LIMIT = (high_limit - close) / close 
# 涨跌停因子 - Python 实现
def calc_limit_factors(df):
    """
    df 需要包含: close, high_limit, low_limit
    数据源: Tushare stk_limit (每日涨跌停价格)
    """
    # 是否触及涨停
    df['hit_up'] = (df['close'] >= df['high_limit'] * 0.998).astype(int)

    # 近20日涨停次数
    df['LIMIT_HIT_UP_20'] = df.groupby('stock_id')['hit_up'].transform(
        lambda x: x.rolling(20).sum()
    )

    # 距涨停距离 (越近越"热")
    df['DIST_TO_LIMIT'] = (df['high_limit'] - df['close']) / df['close']

    # 是否触及跌停
    df['hit_down'] = (df['close'] <= df['low_limit'] * 1.002).astype(int)
    df['LIMIT_HIT_DOWN_20'] = df.groupby('stock_id')['hit_down'].transform(
        lambda x: x.rolling(20).sum()
    )

    return df

4. 北向资金因子

NORTH_RATIO = north_holding / float_share      NORTH_CHG_N = north_holdingt - north_holdingt-N 
# 北向资金因子 - Python 实现
import akshare as ak

def get_north_holding():
    """
    数据源选项:
    1. Tushare: hsgt_top10 (十大活跃股) 或 ggt_monthly (月度持仓)
    2. AkShare: ak.stock_hsgt_hold_stock_em() (每日持仓)
    """
    # AkShare 方式 (免费)
    north_df = ak.stock_hsgt_hold_stock_em(market="北向")
    return north_df

def calc_north_factors(df, north_df):
    # 合并北向持仓数据
    df = df.merge(north_df[['date', 'stock_id', 'north_holding']],
                  on=['date', 'stock_id'], how='left')

    # 北向持仓比例
    df['NORTH_RATIO'] = df['north_holding'] / df['float_share']

    # 5日北向资金变化
    df['NORTH_CHANGE_5'] = df.groupby('stock_id')['north_holding'].transform(
        lambda x: x - x.shift(5)
    )

    # 20日北向资金动量
    df['NORTH_MOM_20'] = df.groupby('stock_id')['north_holding'].transform(
        lambda x: x.rolling(20).apply(
            lambda s: (s.iloc[-1] - s.iloc[0]) / (s.iloc[0] + 1e-8)
        )
    )
    return df

5. 特质波动率因子

ri,t = α + β × rmarket,t + εi,t      IVOL = Std(ε, 20) 
# 特质波动率 - Python 实现
import statsmodels.api as sm

def calc_ivol(df, market_ret, window=20):
    """
    计算特质波动率 (Idiosyncratic Volatility)
    df: 包含 stock_id, date, return 的 DataFrame
    market_ret: 市场收益率 Series (如沪深300日收益率)
    """
    results = []

    for stock_id, group in df.groupby('stock_id'):
        group = group.sort_values('date')
        residuals = []

        for i in range(window, len(group)):
            sub = group.iloc[i - window:i]
            y = sub['return'].values
            X = sm.add_constant(market_ret.loc[sub['date']].values)

            try:
                model = sm.OLS(y, X).fit()
                residuals.append(model.resid.std())
            except:
                residuals.append(np.nan)

        group = group.iloc[window:]
        group['IVOL'] = residuals
        results.append(group)

    return pd.concat(results)

Q7：因子衰减速度有多快？

因子衰减（alpha decay）是指因子的预测能力随时间逐渐减弱的现象。理解衰减速度对因子组合管理和研发资源分配至关重要。

1. 因子衰减的理论框架

α(t) = K / (1 + λt)      双曲线衰减模型（Lee 2025）

Lee 2025 论文提出了因子衰减的双曲线模型，将因子分为两类：

因子类型定义衰减模式拟合度 R²代表因子
机械因子 基于价格/量的统计规律 符合双曲线衰减 ~0.65 动量、反转、波动率
判断型因子 基于基本面/经济逻辑 不符合模型 — 持续有效 低 价值、质量、盈利能力
核心洞察：机械因子需要不断更新，判断型因子可以长期持有 机械因子（如反转、动量）的 alpha 来源是市场微观结构缺陷，随着越来越多的交易者利用这些缺陷，alpha 会被逐渐套利掉。而判断型因子（如价值、质量）的 alpha 来源是人类行为偏差（如锚定效应、损失厌恶），这些偏差几乎不会消失。

2. 各因子的实际衰减速度

因子半衰期衰减类型维护需求说明
短期反转 6-12 个月 双曲线 高 需要持续重算和调整窗口参数
动量/趋势 3-6 个月 双曲线 高 高频交易和量化策略同质化加速衰减
价值因子 30+ 年 几乎不衰减 低 有周期性但长期持续有效
质量因子 30+ 年 不衰减 低 溢价幅度有波动但方向不变
换手率 12-24 个月 缓慢衰减 中 较稳定，与市场结构变化相关
ML挖掘因子 1-3 个月 快速衰减 极高 容易被反向工程，需要持续挖掘新因子
ML挖掘因子的衰减陷阱 通过数据挖掘（如遗传算法、LLM因子挖掘）发现的因子衰减最快，半衰期可能只有 1-3 个月。原因是这些因子往往是对市场噪声的过拟合，或者很快被其他量化团队发现并套利。需要持续保持"因子研发"带宽。

3. 因子拥挤度指标

因子拥挤（factor crowding）是因子衰减的主要驱动力。当过多资金追逐同一因子时，因子溢价被压缩，甚至可能出现崩溃。以下指标可以监控拥挤度：

指标计算方式警戒阈值说明
因子估值价差 多头组与空头组的估值差距 历史最高 10% 价差越窄 = 越拥挤
因子动量 因子近期表现 (短期预测长期) 连续 3 月正收益 表现太好反而是危险信号
策略持仓相似度 相关系数 with factor ETF AUM ρ = -0.63 资金涌入后因子表现反向
因子波动率 因子收益的滚动标准差 高于历史均值 2σ 波动率上升 = 拥挤解除的前兆
拥挤的反转因子崩溃概率高 1.7-1.8 倍 当反转因子出现拥挤信号时（因子动量强 + 策略持仓集中），崩溃概率比正常时期高 1.7-1.8 倍。2024年初的量化踩踏就是一个典型案例：大量量化策略同时做多小盘/反转因子，在流动性收缩时集体平仓。

4. 应对策略

策略一：多因子混合

组合不同衰减周期的因子，降低单因子衰减风险。将快衰减因子（反转、ML因子）和慢衰减因子（价值、质量）混合使用。

策略二：定期重新验证

# 季度/半年度因子有效性检验
def validate_factor(factor_ic_series, min_ic=0.02, min_ir=0.3):
    """
    每季度检查因子是否仍然有效
    """
    recent_ic = factor_ic_series.tail(60)  # 近60个交易日
    ic_mean = recent_ic.mean()
    ic_std = recent_ic.std()
    ir = ic_mean / (ic_std + 1e-8)

    status = "ACTIVE"
    if abs(ic_mean) < min_ic:
        status = "DEGRADED"  # IC 过低，考虑降权
    if abs(ir) < min_ir:
        status = "UNSTABLE"  # IR 过低，信号不稳定

    return {
        "ic_mean": ic_mean,
        "ir": ir,
        "status": status
    }

策略三：监控 IC 滚动均值和标准差

  • 滚动 IC 均值持续下降 → 因子正在衰减
  • 滚动 IC 标准差上升 → 因子变得不稳定
  • IC 符号反转 → 因子已失效，需立即移除

策略四：预留因子研发带宽

建议将 20-30% 的研发时间用于新因子探索，而非仅仅维护现有因子。特别关注：

  • 另类数据因子（新闻情绪、卫星数据、供应链数据）
  • 高频日内聚合因子（波峰因子、VWAP偏离等）
  • LLM辅助的因子表达式优化

Q8：如何做因子中性化？

因子中性化是将因子值中与某些已知风险源（行业、市值等）相关的部分剥离，只保留纯粹的 alpha 信号。这是从因子研究到实盘策略的关键步骤。

1. 为什么需要中性化

问题举例不中性化的后果
行业估值差异 银行 PE=5，科技 PE=50 PE 因子会永远做多银行做空科技
市值干扰 小市值天然 PE 低、波动高、换手率高 多个因子实际上都在做多小盘股
风险集中 组合集中在某些行业/市值区间 非因子驱动的系统性风险暴露
不中性化 = 伪 alpha 如果一个因子与市值相关系数高达 0.7，那么它的"alpha"本质上是小盘股溢价。一旦小盘股风格反转（如 2024 年踩踏），所有"alpha"会同时消失。中性化让你确认：因子的预测能力是否独立于已知风险因素。

2. 行业中性化 (Industry Neutralization)

方法：在每个行业内分别做 Z-score 标准化，使因子值在行业内可比。

factor_neutrali = (factori - meanindustry) / stdindustry 
# 行业中性化 - Python 实现
def industry_neutralize(df, factor_col, industry_col='sw_industry_1'):
    """
    在每个交易日的每个行业内做 Z-score 标准化
    行业分类: 申万一级 (31个行业) 最常用
    """
    df[f'{factor_col}_ind_neutral'] = df.groupby(
        ['date', industry_col]
    )[factor_col].transform(
        lambda x: (x - x.mean()) / (x.std() + 1e-8)
    )
    return df

# 使用示例
df = industry_neutralize(df, 'EP')
# 现在 EP_ind_neutral 在行业内可比
# 银行股之间比EP有意义，银行和科技之间比EP无意义

Qlib 实现：Qlib 的 CSRankNorm 处理器可以做截面排名标准化，虽然不是严格的行业中性化，但效果类似。若需严格行业中性化，需自定义 DataHandler。

3. 市值中性化 (Size Neutralization)

方法：对 ln(市值) 做截面回归，取残差作为中性化后的因子值。

factori = α + β × ln(mcapi) + εi      factor_neutrali = εi 
# 市值中性化 - Python 实现
import numpy as np
from sklearn.linear_model import LinearRegression

def size_neutralize(df, factor_col, mcap_col='market_cap'):
    """
    对每个交易日做截面回归，剥离市值影响
    很多因子与市值高度相关:
    - 小市值天然低PE、高换手率、高波动
    - 不中性化 = 多个因子都在赌小盘
    """
    def _neutralize(group):
        ln_mcap = np.log(group[mcap_col].values).reshape(-1, 1)
        factor = group[factor_col].values

        # 剔除 NaN
        mask = ~(np.isnan(ln_mcap.ravel()) | np.isnan(factor))
        if mask.sum() < 10:
            return pd.Series(np.nan, index=group.index)

        reg = LinearRegression()
        reg.fit(ln_mcap[mask], factor[mask])
        residual = factor - reg.predict(ln_mcap)
        return pd.Series(residual, index=group.index)

    df[f'{factor_col}_size_neutral'] = df.groupby('date').apply(
        _neutralize
    ).reset_index(level=0, drop=True)
    return df

4. 联合中性化（行业 + 市值）

factor = α + Σ(βi × Industryi) + γ × ln(mcap) + ε      ε = 中性化后因子值 
# 联合中性化 (行业 + 市值) - 最完整版本
import pandas as pd
import numpy as np
from sklearn.linear_model import LinearRegression

def joint_neutralize(df, factor_col, industry_col='sw_industry_1',
                      mcap_col='market_cap'):
    """
    行业哑变量 + ln(市值) 做截面回归, 取残差
    这是最标准的因子中性化方法
    """
    def _neutralize(group):
        # 构建行业哑变量
        dummies = pd.get_dummies(group[industry_col], drop_first=True)

        # 构建回归矩阵: [行业哑变量, ln(市值)]
        X = pd.concat([
            dummies,
            np.log(group[mcap_col])
        ], axis=1)

        y = group[factor_col]

        # 剔除 NaN
        mask = ~(X.isna().any(axis=1) | y.isna())
        if mask.sum() < 30:
            return pd.Series(np.nan, index=group.index)

        reg = LinearRegression()
        reg.fit(X[mask], y[mask])
        residual = y - reg.predict(X)
        return pd.Series(residual.values, index=group.index)

    df[f'{factor_col}_neutral'] = df.groupby('date').apply(
        _neutralize
    ).reset_index(level=0, drop=True)
    return df

5. RobustZScoreNorm vs ZScoreNorm

方法公式优点缺点适用场景
ZScoreNorm (x - mean) / std 简单直观 受极端值影响大 数据分布接近正态时
RobustZScoreNorm (x - median) / MAD 抗极端值 计算略慢 金融数据（厚尾分布）
CSRankNorm 截面排名 → [0, 1] 完全消除极端值 损失绝对水平信息 树模型 / 非线性模型

其中 MAD = Median Absolute Deviation = median(|x - median(x)|)。Qlib 默认使用 RobustZScoreNorm，这是金融数据处理的最佳实践。

Qlib 推荐：RobustZScoreNorm + CSRankNorm 双层标准化 先用 RobustZScoreNorm 做初步标准化（去极端值），再用 CSRankNorm 做截面排名。这样既保留了因子的序数信息，又完全消除了分布形状的影响。树模型（如 LightGBM）尤其受益于排名标准化。

6. 截面排名标准化 (CSRankNorm)

# CSRankNorm 实现
def cs_rank_norm(df, factor_col):
    """
    截面排名标准化: 将因子值转化为截面排名的百分位 [0, 1]
    - 消除极端值影响
    - 使分布均匀 (uniform distribution)
    - 不同因子之间变得可比
    """
    df[f'{factor_col}_rank'] = df.groupby('date')[factor_col].transform(
        lambda x: x.rank(pct=True)
    )
    return df

Step 1

Step 1: 原始因子值

直接计算得到的因子原始值，可能包含极端值、行业偏差、市值偏差。

Step 2: 去极值 (Winsorize)

# 去极值: 3σ 法 或 MAD 法
def winsorize_mad(series, n_mad=5):
    """MAD 方法去极值 (更稳健)"""
    median = series.median()
    mad = (series - median).abs().median()
    upper = median + n_mad * mad
    lower = median - n_mad * mad
    return series.clip(lower, upper)

def winsorize_sigma(series, n_sigma=3):
    """3σ 方法去极值 (传统方法)"""
    mean = series.mean()
    std = series.std()
    upper = mean + n_sigma * std
    lower = mean - n_sigma * std
    return series.clip(lower, upper)

# 对每个交易日截面去极值
df['factor_win'] = df.groupby('date')['factor'].transform(winsorize_mad)

Step 3: 行业 + 市值中性化

使用前文的 joint_neutralize() 函数。

Step 4: 标准化

使用 RobustZScoreNorm 或 CSRankNorm（推荐树模型使用后者）。

中性化的权衡 中性化不是免费的。过度中性化（同时中性化行业 + 市值 + 波动率 + ...）会剥离掉有价值的信号。实践中推荐仅做行业 + 市值联合中性化，保留其他维度的暴露让模型自行学习。树模型（LightGBM）对因子分布不敏感，CSRankNorm 后通常不需要额外的行业中性化。

9. 实践建议总结

场景推荐 Pipeline原因
树模型 (LightGBM) Winsorize → CSRankNorm 树模型只看排名，不需要严格中性化
线性模型 Winsorize → 行业+市值中性化 → ZScore 线性模型对多重共线性敏感，需要中性化
深度学习模型 Winsorize → RobustZScore → CSRankNorm 神经网络需要标准化输入，排名有助于稳定训练
因子研究/IC 检验 Winsorize → 行业+市值中性化 → ZScore 确认纯 alpha 信号，排除行业/市值混淆
最终建议 对于 Qlib + LightGBM 的典型场景，使用 Qlib 默认的 RobustZScoreNorm + Fillna + CSRankNorm pipeline 即可满足大部分需求。只有在因子研究阶段（用 alphalens 做 IC 检验）才需要额外的行业 + 市值联合中性化，以确认因子的独立 alpha 贡献。
