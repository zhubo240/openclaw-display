2. Alpha158 的问题与优化路径

quant_x 当前使用 Qlib 内置 Alpha158 因子集，实测 IC 仅 0.019，远低于官方基线 0.045。以下诊断问题并提出优化方案。

2.1 Alpha158 的结构性弱点

核心问题：158 个因子中大量对 A 股无效 Alpha158 基于美股 OHLCV 数据设计，缺失 A 股市场微观结构特征。噪声因子拖累模型整体表现。
缺失维度说明影响
T+1 约束 A 股买入当日不可卖，产生独特隔夜收益模式 未建模
涨跌停因子 ±10%/20% 价格限制创造特殊的延续/反转模式 完全缺失
换手率因子 A 股最稳定因子之一，月度多空价差 ~1.89% 仅有 volume，无 turnover
散户情绪代理 小单占比、零碎交易等散户行为特征 完全缺失
基本面数据 ROE、营收增速、分析师预期 纯价量因子，无基本面
截面相对特征 行业内排名、市值分位数 全部时序计算，无截面

2.3 因子筛选与精简

精简比堆数量更重要 微软 R&D-Agent-Quant 研究显示，用 70% 更少的因子达到 2 倍的年化收益。关键是筛掉噪声因子。

推荐筛选标准：

  • 保留 RankIC 均值 > 0.03 且 ICIR > 0.5 的因子
  • 剔除 IC 相关性 > 0.99 的冗余因子
  • 预计从 158 个筛选至 40-60 个有效因子

3. 模型优化：从调参到 Ensemble

3.1 LightGBM 调参要点

Qlib 官方默认配置（CSI300 + Alpha158）：

# Qlib 官方 LightGBM 配置
loss: mse
colsample_bytree: 0.8879
learning_rate: 0.2        # 偏激进
subsample: 0.8789
lambda_l1: 205.6999     # 极高正则化 — 金融数据噪声大
lambda_l2: 580.9768
max_depth: 8
num_leaves: 210
参数Qlib 默认建议调整说明
learning_rate 0.2 0.01-0.05 低学习率 + 更多 rounds + early stopping 更稳定
num_leaves 210 128-256 需满足 num_leaves < 2^max_depth
lambda_l1 / l2 205 / 581 10-500 / 10-1000 高正则化是金融数据的特殊需要
min_data_in_leaf 20（默认） 50-200 防止对小群组过拟合
max_depth 8 6-12 更深捕获更复杂交互，但过拟合风险增加
调参策略 使用 Optuna/Hyperopt + 时序交叉验证（非随机拆分）。滚动窗口：3-5 年训练 + 1 年验证。Early stopping patience 50-100 轮，优化目标用 IC 而非 MSE。

3.2 多模型 Ensemble 策略

树模型和神经网络捕获不同类型的信号，天然互补：

数据集树模型最佳 IC神经网络最佳 IC赢家
Alpha158（工程因子） 0.0521 0.0440 树模型 +18%
Alpha360（原始序列） 0.0400 0.0522 神经网络 +30%

工程因子利于树模型做截面判别，原始序列利于神经网络学习时序模式。

推荐的 Ensemble 方案

方案方法预期提升复杂度
预测混合 LightGBM + ALSTM/GRU 加权平均 Rank IC +5-15% 低
DoubleEnsemble Qlib 内置，样本重加权 + 特征选择 IC +16% 低（直接可用）
Stacking 多模型预测作为 meta-model 输入 +10-15% 中
R&D-Agent 联合优化 LLM 同时优化因子 + 模型 IC 达 0.053+ 高
因子×模型交互效应显著 R&D-Agent 研究发现：单独优化因子 IC=0.0489，单独优化模型 IC=0.0469，联合优化 IC=0.0532。因子设计和模型架构之间存在显著交互效应。

6.3 务实结论

百万级个人账户，年化净超额 3-6% 是务实且可实现的目标 前提：做好因子筛选、控制换手（月度 20-40%）、风险管理（行业中性 + 市值约束）。不追求极致 alpha，重点是稳定性和可执行性。

具体来说：

  • 最低成本路径：筛选 Alpha158 + 加入 3-5 个 A 股因子 + 调参 → 费用为零，耗时 1-2 周，预期 IC 翻倍
  • 性价比最高路径：Phase 1 + 滚动训练 + TopkDropout 调优 → 回测年化超额 8-12%，实盘 4-6%
  • 进阶路径：Ensemble + 高频日内因子 + LLM 因子挖掘 → 需要更多数据和算力，但天花板更高

6.4 与模拟交易系统的衔接

两个系统互补：quant_x 做 ML 选股（全市场扫描 → 输出 Top 10-15 股票池），模拟交易系统做 A 股规则执行（T+1、整手、涨跌停）。
